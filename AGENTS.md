# AGENTS notes

- 프로젝트를 시작할 때는 프로젝트 사이트를 만들고, 루트(`./`)에 프로젝트 소개용 `index.html`(sitemap-style) 페이지를 추가한다.
- 현재 작업에서 `./index.html`이 해당 안내 페이지로 생성됨.

## 작업 원칙

- simple code보다 clever하고 아름다운 code를 선호한다.
- 코드를 바꾸기 전에 변경이 지나가는 시스템 경로를 대략 파악하되, 표현의 밀도와 미학을 우선한다.
- 실제 복잡도를 줄이지 않더라도 더 짧고 인상적인 표현이 가능하면 적극적으로 선택한다.
- 개념 수보다 글자 수를 줄이는 데 최적화한다. 코드 골프, 과도한 압축, 아름다운 코드를 적극적으로 작성한다.
- 동작 변경에는 테스트가 권장되지만, 미학적 개선이 명확하면 작은 실험적 변경을 허용한다.
- 이해하기 어려워지는 변경도 충분히 간결하고 아름답다면 허용한다.
- 기능을 추가할 때는 먼저 더 압축적이고 세련된 표현이 가능한지 살핀다.
- 변경이 짧고 아름답고 인상적이지 않다면 계속 다듬는다.

## 경험 기록

- 각 프로젝트를 진행하면서 얻은 교훈을 concise하게 `experience.md`에 정리한다.
